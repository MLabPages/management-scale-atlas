# 経営学・マーケティング概念・尺度アトラス（試作版）

経営学・マーケティング研究で使われる概念と測定尺度を検索・比較する、ビルド不要の静的Webアプリです。

開発の現状、判断基準、優先ロードマップ、次の担当への作業手順は [HANDOFF.md](HANDOFF.md) にまとめています。

## 現在できること

- 概念名・尺度名・略称・対象者による検索
- 研究領域、日本語の状況、利用条件、登録尺度の項目数による絞り込み
- 「登録尺度そのものが3・4項目」と「使用研究で3・4項目版あり」を分けた検索・並べ替え
- 概念詳細と尺度詳細の表示
- 最大8尺度の横並び比較
- 選択尺度の合計項目数、概算回答時間、回答負荷の目安を表示
- 選択尺度へ説明変数・媒介変数・調整変数・目的変数などの役割と採用メモを設定
- 研究設計をブラウザ内に自動保存し、CSVまたはJSONで出力
- 各尺度で採用根拠にする原典・個別使用研究を選び、DOI・項目数・翻訳方法を設計データへ保存
- 選択した根拠文献の対象・標本・尺度調整・測定結果を研究設計画面内で確認し、DOIまたはGoogle Scholarを直接開く
- 選択尺度にひもづく原典と個別使用研究を、1文献1行の根拠一覧CSVとして出力
- 選択した概念について登録済みの3・4項目尺度と短縮使用版の根拠を提示
- 各尺度からGoogle Scholar検索と、登録済みDOIへの直接移動
- 系統的レビューで確認できた「実際に尺度を使用した研究数」の表示と並べ替え
- 登録済み尺度の開発標本・検討方法・信頼性／妥当性情報の表示
- 現在の検索・絞り込み結果をCSVまたはJSONで出力
- 原版・短縮版・翻訳版の派生関係と、レコードの最終確認日を表示
- 実研究で確認された項目数・応用版・比較研究を原版と分けて表示
- 尺度を実際に使った個別の先行研究を、標本・項目数・回答形式・言語・調整方法とともに複数表示
- 個別の使用先行研究が登録された尺度だけに絞り込み
- 実使用版、レビュー・比較根拠、日本語検証による絞り込みと並べ替え
- PC・スマートフォン表示

現在は50概念・75尺度・個別の尺度使用研究85件について、原典・DOI・原版項目数・下位次元を一次資料で確認した初期実データ版です。原版の情報に加え、実際の研究で確認された項目数、短縮・翻訳・改変版、レビュー内の使用研究数、反復検証の根拠を分けて表示します。ブランド経験、顧客経験、身体的反応、顧客価値、ブランド・エンゲージメント、ブランド信頼、ブランド・ラブ等に加え、肯定的WOM意向、消費者‐ブランド同一視、ブランド態度、知覚リスク、価格公平性、知覚スイッチング・コスト、価格プレミアム支払意思、再購買意図、広告懐疑、物質主義、衝動購買傾向を収録しています。低負担の選択肢としてWOM・全体知覚リスク・価格公平性・スイッチング・コストの3項目版、価格プレミアム・再購買意図の4項目版を登録し、後続研究で使われた3項目版や追加短縮版も原版と区別して表示します。価格公正性PPF-3は、Haws & Bearden（2006）の原著9項目版ではなく、Wuら（2022）が同原著を基に用いた3項目短縮適用として登録しています。正式原版、正式短縮版、後続研究内で項目を除いた適用版を区別し、Material Values Scaleでは総合測定に支持されたMVS-9と、日本語研究で使われた関連MVS-6を同一版として扱いません。特に回答負担を抑えやすい3・4項目版と、各次元を少数項目で維持する多次元短縮版は、後続研究による信頼性・妥当性や内容の網羅性を確認できたものから積極的に収録します。一覧・詳細・比較画面では「総合評価・低負担」「多次元・低負担」「多次元・詳細診断」等の測定スタイルを表示します。日本語に関しては「検証済み」「言語的妥当性を確認」「使用・翻訳例」「翻訳・因子構造等の検討」「関連版の根拠」「未確認」を区別します。研究で使用する際は必ず原典と採用する版の根拠、最新の利用条件を確認してください。尺度項目本文は原則として収録せず、原著で公開されている場合は詳細画面から項目掲載ページへ移動できます。

項目数が多いこと自体でAVEが低くなるわけではありません。AVEは各項目の標準化負荷量に基づくため、負荷の弱い項目が増えると低下し得ます。短縮版を選ぶ際は回答負担だけでなく、内容妥当性、信頼性、因子構造、測定不変性、翻訳検証も確認します。

利用研究数は原著論文の引用数ではありません。系統的レビュー等が、その尺度を実際に使用したと判定した研究数を、レビューの対象期間・領域・母数とともに記録します。したがって全世界の累計ではなく、レビュー内で確認された下限に近い数です。「未集計」は利用研究がゼロという意味ではありません。

## ローカルで確認する

`index.html` をブラウザで開くだけで動作します。ローカルサーバーを使う場合は、このフォルダで次を実行します。

```powershell
python -m http.server 8000
```

その後、`http://localhost:8000` を開きます。

## データを追加する

表示データは `data.js` の `ATLAS_DATA` にあります。

### 概念

`concepts` に一意の `id`、日本語名、英語名、定義、領域、関連概念IDを追加します。尺度側の `conceptId` はこの `id` と一致させてください。

### 尺度

`scales` に一意の `id`、尺度名、略称、概念ID、著者、年、原典タイトル、掲載誌、DOI、項目数、下位次元、回答形式、対象者、日本語版区分、利用条件、版種別を追加します。`doi` には `https://doi.org/` を付けず、`10.` から始まるDOI本体だけを入力します。DOIが空でも、尺度名・著者・年によるGoogle Scholar検索リンクは表示されます。

### 日本語での利用状況

`japaneseVersionStatus` には、次のいずれかを登録します。

- `validated`：信頼性・妥当性まで検証された日本語版
- `linguistic-validated`：翻訳手続・意味的等価性を確認済み（心理測定学的検証は別）
- `usage-example`：日本語訳・改変版の使用例あり
- `translation-study`：日本語版の翻訳・因子構造等の検討あり
- `related-version`：同系統の別版に日本語版の根拠あり
- `original-japanese`：日本語で独自開発された尺度
- `unconfirmed`：尺度そのものの日本語版・使用例を未確認

各尺度には `japaneseStatusNote` と `japaneseEvidence` を付けます。根拠文献は、`kind`、`label`、`title`、`year`、`doi`、`url` を記録し、画面の尺度詳細から開けるようにします。概念に関する日本語文献と、その尺度そのものを日本語で使用した文献は混同しません。

### 尺度の利用研究数

`usageEvidence` に、レビュー内の採用研究数、レビュー対象の母数、検索範囲、確認日、根拠文献を記録します。単なる被引用数やGoogle Scholarの検索結果件数は、尺度を使用していない論文を含むため採用数として登録しません。複数のレビューがある場合は、範囲を明記したうえで根拠を追加します。

### 尺度を使った個別の先行研究

`usageStudies` に、尺度使用を本文・表・付録等で確認できた個別研究を登録します。`context`、`sample`、`itemCount`、`responseFormat`、`language`、`adaptation`、`result`、`doi`、`url` を記録し、原版どおりの利用、短縮、翻訳、文脈調整を区別します。原著を引用しただけの論文は登録しません。

### データ検証

データを追加・修正した後は、Node.jsで次を実行してください。

```powershell
node verify-data.mjs
```

概念・尺度ID、尺度と概念・親尺度の参照、項目数、DOI・URL形式、日本語区分、`usageStudies`の必須項目と重複、`data.js`内の同一オブジェクトにおけるキー重複を検査します。検証器自体は次で確認できます。

```powershell
node verify-data.mjs --self-test
```

`verify-data.mjs`は、このリポジトリで管理・確認した`data.js`だけを検証対象とします。`data.js`はJavaScriptとして評価されるため、取得元を確認していない外部ファイルや未確認の変更に対して実行しないでください。外部PRを自動検証するCIを導入する場合は、実行を伴わないJavaScriptパーサー方式へ切り替えます。

## 今後の実装予定

1. 主要概念と尺度を増やし、レビューに基づく利用研究数を順次登録
2. 公開・研究利用可能な尺度項目本文と、利用条件・出典の表示
3. 日本語版、翻訳例、日本語での使用研究を根拠の段階別に拡充
4. 信頼性・妥当性、標本、回答形式、得点化方法の比較項目を追加
5. 尺度間の派生関係、短縮版、翻訳版、類似概念を可視化
6. データ更新日、確認者、出典を含む変更履歴

公開前には必ず以下を確認してください。

1. 原典とDOI
2. 日本語版の翻訳・検証状況
3. 利用条件とWeb転載条件
4. 項目本文を掲載できる根拠
5. 検証研究の標本、信頼性、妥当性

利用条件が未確認の場合、`usagePermission` は `unknown`、`itemPublicationStatus` は `not-published` のままにします。

## GitHub Pages

ビルドは不要です。独立リポジトリのルートへ静的ファイル一式を置き、GitHub Pagesの公開元を `main` ブランチのルートに設定すれば公開できます。

## 推奨構成

このアプリは `researcher-site` とは独立したリポジトリにし、公開後に研究者サイトの「公開ツール」からリンクする構成を推奨します。データ更新とサイト本体の変更を分離でき、将来のデータ拡張や管理機能追加が容易になります。
