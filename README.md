# 経営学・マーケティング概念・尺度アトラス（試作版）

経営学・マーケティング研究で使われる概念と測定尺度を検索・比較する、ビルド不要の静的Webアプリです。

## 現在できること

- 概念名・尺度名・略称・対象者による検索
- 研究領域、日本語の状況、利用条件、項目数（3・4項目以下を含む）による絞り込み
- 概念詳細と尺度詳細の表示
- 最大4尺度の横並び比較
- 各尺度からGoogle Scholar検索と、登録済みDOIへの直接移動
- 系統的レビューで確認できた「実際に尺度を使用した研究数」の表示と並べ替え
- 登録済み尺度の開発標本・検討方法・信頼性／妥当性情報の表示
- 現在の検索・絞り込み結果をCSVまたはJSONで出力
- 原版・短縮版・翻訳版の派生関係と、レコードの最終確認日を表示
- 実研究で確認された項目数・応用版・比較研究を原版と分けて表示
- 実使用版、レビュー・比較根拠、日本語検証による絞り込みと並べ替え
- PC・スマートフォン表示

## 重要な注意

現在は30概念・40尺度について、原典・DOI・原版項目数・下位次元を一次資料で確認した初期実データ版です。原版の情報に加え、実際の研究で確認された項目数、短縮・翻訳・改変版、レビュー内の使用研究数、反復検証の根拠を分けて表示します。特に回答負担を抑えやすい3・4項目版は、後続研究による信頼性・妥当性や内容の網羅性を確認できたものから積極的に収録します。日本語に関しては「検証済み」「言語的妥当性を確認」「使用・翻訳例」「関連版の根拠」「未確認」を区別します。研究で使用する際は必ず原典と採用する版の根拠、最新の利用条件を確認してください。尺度項目本文は収録していません。

項目数が多いこと自体でAVEが低くなるわけではありません。AVEは各項目の標準化負荷量に基づくため、負荷の弱い項目が増えると低下し得ます。短縮版を選ぶ際は回答負担だけでなく、内容妥当性、信頼性、因子構造、測定不変性、翻訳検証も確認します。

利用研究数は原著論文の引用数ではありません。系統的レビュー等が、その尺度を実際に使用したと判定した研究数を、レビューの対象期間・領域・母数とともに記録します。したがって全世界の累計ではなく、レビュー内で確認された下限に近い数です。「未集計」は利用研究がゼロという意味ではありません。

## ローカルで確認する

`index.html` をブラウザで開くだけで動作します。ローカルサーバーを使う場合は、このフォルダで次を実行します。

```powershell
python -m http.server 8000
```

その後、`http://localhost:8000` を開きます。

## データを追加する

表示データは `data.js` の `ATLAS_DATA` にあります。

### 概念

`concepts` に一意の `id`、日本語名、英語名、定義、領域、関連概念IDを追加します。尺度側の `conceptId` はこの `id` と一致させてください。

### 尺度

`scales` に一意の `id`、尺度名、略称、概念ID、著者、年、原典タイトル、掲載誌、DOI、項目数、下位次元、回答形式、対象者、日本語版区分、利用条件、版種別を追加します。`doi` には `https://doi.org/` を付けず、`10.` から始まるDOI本体だけを入力します。DOIが空でも、尺度名・著者・年によるGoogle Scholar検索リンクは表示されます。

### 日本語での利用状況

`japaneseVersionStatus` には、次のいずれかを登録します。

- `validated`：信頼性・妥当性まで検証された日本語版
- `linguistic-validated`：翻訳手続・意味的等価性を確認済み（心理測定学的検証は別）
- `usage-example`：日本語訳・改変版の使用例あり
- `translation-study`：日本語版の翻訳・因子構造等の検討あり
- `related-version`：同系統の別版に日本語版の根拠あり
- `original-japanese`：日本語で独自開発された尺度
- `unconfirmed`：尺度そのものの日本語版・使用例を未確認

各尺度には `japaneseStatusNote` と `japaneseEvidence` を付けます。根拠文献は、`kind`、`label`、`title`、`year`、`doi`、`url` を記録し、画面の尺度詳細から開けるようにします。概念に関する日本語文献と、その尺度そのものを日本語で使用した文献は混同しません。

### 尺度の利用研究数

`usageEvidence` に、レビュー内の採用研究数、レビュー対象の母数、検索範囲、確認日、根拠文献を記録します。単なる被引用数やGoogle Scholarの検索結果件数は、尺度を使用していない論文を含むため採用数として登録しません。複数のレビューがある場合は、範囲を明記したうえで根拠を追加します。

## 今後の実装予定

1. 主要概念と尺度を増やし、レビューに基づく利用研究数を順次登録
2. 公開・研究利用可能な尺度項目本文と、利用条件・出典の表示
3. 日本語版、翻訳例、日本語での使用研究を根拠の段階別に拡充
4. 信頼性・妥当性、標本、回答形式、得点化方法の比較項目を追加
5. 尺度間の派生関係、短縮版、翻訳版、類似概念を可視化
6. データ更新日、確認者、出典を含む変更履歴

公開前には必ず以下を確認してください。

1. 原典とDOI
2. 日本語版の翻訳・検証状況
3. 利用条件とWeb転載条件
4. 項目本文を掲載できる根拠
5. 検証研究の標本、信頼性、妥当性

利用条件が未確認の場合、`usagePermission` は `unknown`、`itemPublicationStatus` は `not-published` のままにします。

## GitHub Pages

ビルドは不要です。独立リポジトリのルートへ5ファイルを置き、GitHub Pagesの公開元を `main` ブランチのルートに設定すれば公開できます。

## 推奨構成

このアプリは `researcher-site` とは独立したリポジトリにし、公開後に研究者サイトの「公開ツール」からリンクする構成を推奨します。データ更新とサイト本体の変更を分離でき、将来のデータ拡張や管理機能追加が容易になります。
