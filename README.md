# 経営学・マーケティング概念・尺度アトラス（試作版）

経営学・マーケティング研究で使われる概念と測定尺度を検索・比較する、ビルド不要の静的Webアプリです。

## 現在できること

- 概念名・尺度名・略称・対象者による検索
- 研究領域、日本語版、利用条件、項目数による絞り込み
- 概念詳細と尺度詳細の表示
- 最大4尺度の横並び比較
- 各尺度からGoogle Scholar検索と、登録済みDOIへの直接移動
- PC・スマートフォン表示

## 重要な注意

現在は5概念・8尺度について、原典・DOI・項目数・下位次元を一次資料で確認した初期実データ版です。利用条件と日本語版は、確認できないものを推測せず「未確認」「なし」としています。研究で使用する際は必ず原典と最新の利用条件を確認してください。尺度項目本文は収録していません。

## ローカルで確認する

`index.html` をブラウザで開くだけで動作します。ローカルサーバーを使う場合は、このフォルダで次を実行します。

```powershell
python -m http.server 8000
```

その後、`http://localhost:8000` を開きます。

## データを追加する

表示データは `data.js` の `ATLAS_DATA` にあります。

### 概念

`concepts` に一意の `id`、日本語名、英語名、定義、領域、関連概念IDを追加します。尺度側の `conceptId` はこの `id` と一致させてください。

### 尺度

`scales` に一意の `id`、尺度名、略称、概念ID、著者、年、原典タイトル、掲載誌、DOI、項目数、下位次元、回答形式、対象者、日本語版区分、利用条件、版種別を追加します。`doi` には `https://doi.org/` を付けず、`10.` から始まるDOI本体だけを入力します。DOIが空でも、尺度名・著者・年によるGoogle Scholar検索リンクは表示されます。

公開前には必ず以下を確認してください。

1. 原典とDOI
2. 日本語版の翻訳・検証状況
3. 利用条件とWeb転載条件
4. 項目本文を掲載できる根拠
5. 検証研究の標本、信頼性、妥当性

利用条件が未確認の場合、`usagePermission` は `unknown`、`itemPublicationStatus` は `not-published` のままにします。

## GitHub Pages

ビルドは不要です。独立リポジトリのルートへ5ファイルを置き、GitHub Pagesの公開元を `main` ブランチのルートに設定すれば公開できます。

## 推奨構成

このアプリは `researcher-site` とは独立したリポジトリにし、公開後に研究者サイトの「公開ツール」からリンクする構成を推奨します。データ更新とサイト本体の変更を分離でき、将来のデータ拡張や管理機能追加が容易になります。
